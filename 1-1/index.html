This guide helps architects and engineering leads select the right cloud platform and design system for their projects. It serves as the foundational decision-making resource before diving into specific UI tooling (see 1-2) or implementation strategies (see 1-3).

Design System Architecture¶

Why Design Systems Matter¶

A design system bridges the gap between design and development, ensuring consistency across applications. Modern design systems are:

  • Cross-platform: Work across web, mobile, desktop

  • Design-to-code: Integrated workflows from design tools (Figma) to component libraries

  • Token-based: Separation of design tokens from implementation

Blazor Design System Strategy (Enterprise Focus)¶

Microsoft Azure teams commonly use Storybook to host CSS/Sass and design tokens while building a separate Blazor component library that consumes those styles. This hybrid approach combines the best of both worlds:

Workflow:

Figma Design → Storybook (CSS/Design Tokens) ↔ Figma<>Storybook Plugin
                                              ↓
                                    Blazor Component Library
                                              ↓
                                    RCL (Razor Class Library)
                                              ↓
                                    Multiple Projects

Key Integration Tool: Storybook Connect Plugin allows you to link live stories directly to Figma components, creating a bidirectional design-to-code connection.

Cross-Reference: JPL’s Explorer 1 Design System demonstrates an alternative approach primarily built with React and TypeScript, supporting React, Vue, and Web Components rather than Blazor.

Blazor Component Libraries¶

  • MudBlazor: Open-source, Material Design-based library. Highly customizable with built-in theming support. Popular for modern enterprise UIs.

  • Fluent UI: Microsoft’s official implementation of the Fluent Design System. Ensures consistency with modern Microsoft applications.

  • Radzen Blazor: 70+ free components with Radzen Blazor Studio, a WYSIWYG designer for visual logic management.

  • Blazorize: Switch between Bootstrap, Bulma, or Material Design CSS frameworks while maintaining consistent C# code.

Recommended Blazor Implementation Strategy¶

  1. Host Components in a Razor Class Library (RCL)

    • Enables component reuse across multiple projects

    • Keeps components platform-agnostic

    • Simplifies versioning and distribution

  2. Use Blazing Story for Living Documentation

    • Integrate into your RCL

    • Creates a live catalog for designers and developers

    • Serves as single source of truth

  3. Leverage RenderFragment

    • Create flexible, modular components

    • Support nested content and template hooks

    • Enable composition patterns
